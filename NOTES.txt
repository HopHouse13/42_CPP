NAMESPACE

ensemble de varibles & fonctions qui se trouve dans le meme scoop.
(c'est comme une struct amelioree)

POur les variables globales, il faut declarer + initialiser sur la meme ligne.
Pour declarer une string il faut comme type : const char *[name_de_la_variable].

-operateur "resolution de portee"-
Pour faire appel a une varible d'une namespace.

ENTREE/SORTIE

Pour affiche dans la sortie std ou recuperer dans l'entree srd,
il faut utiliser "std" dans la namespace <iostream>.
Pour la sortie std(printf) : "std::cout" << [le truc] 
Pour l'entree std(scanf) : "std::cint" >> [le truc]

std::endl sert a avoir une retour a la ligne

CLASS/INSTANCE

-class-
declaration d'une class: 

class Erika
{
	public:
		Erika(void);	// constructeur
		~Erika(void);	// destructeur
};

Les constructeurs/destructeur n'ont pas de return.

class -> ~structure/type
instance -> ~variable

Les instances sont detruites a la sortie du scoop de la fonction
ou elles ont ete declarees.

Lors de la declaration de l'instance, le constructeur est automatiqument execute
et lors de la sortie du scoop de la fonction, le destructeur est automatiquement appele.

THIS

"this" est un pointeur sur l'instance courante dans la quelle on est.
Le pointeur de l'instance est transparant. Cela veut dire que qu'il transmit
dans toutes les fonctions du scoop de la fonction ou l'instance a ete declaree.
